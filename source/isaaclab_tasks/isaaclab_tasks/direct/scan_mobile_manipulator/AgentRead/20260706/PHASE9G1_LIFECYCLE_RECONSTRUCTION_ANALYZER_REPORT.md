## 5. Analyzer Behavior

Analyzer path:

```text
scripts/environments/analyze_phase9g1_lifecycle_reconstruction.py
```

The analyzer:

```text
accepts one or more existing assignment_history.csv files
builds a per-file column inventory before reconstruction
reports required/optional columns present and missing
falls back to basic assignment proxy reconstruction when optional columns are missing
emits combined row-level reconstructed lifecycle CSV
emits file-level summary CSV
emits aggregate summary JSON
emits column inventory JSON
prints a concise console summary
```

Output files per run:

```text

```

## 11. Evidence for Phase 9G-2

The existing Phase 9F histories support starting Phase 9G-2 design/implementation for disabled-by-default wrapper-local failed-pair/release memory.

Reason:

```text
12 / 12 budget-failed proxy segments were released and later reacquired by the same owner.
same-owner return delay was consistently 5 steps.
coverage_gain_within_20_after_release was 0 / 12.
Phase 9F-5 redirect guardrail did not reduce this pattern relative to Phase 9F-2C.
```

This is diagnostic evidence only. It does not prove a behavior change will improve performance, and it does not justify training yet. Phase 9G-2 should remain disabled by default, preserve observation/action/reward compatibility, log overmask/noop/fail-open diagnostics, and be validated by fake-env smoke before any playback.
